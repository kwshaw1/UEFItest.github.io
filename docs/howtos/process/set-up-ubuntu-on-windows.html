Set up Ubuntu on Windows for Sphinx development¶

This guide will walk you through the process of installing and configuring everything you need to be productive with git and Sphinx publishing using OTC-TCS workflows.

Install the Windows Subsystem for Linux¶

  1. Open a powershell terminal with administrative privileges and run the following command:

    Enable-WindowsOptionalFeature -Online -FeatureName Microsoft-Windows-Subsystem-Linux

  2. Click the link to open the Microsoft store and get the Ubuntu disro: https://www.microsoft.com/store/p/ubuntu/9nblggh4msv6

  3. Launch the Ubuntu app from the start menu and follow the prompts to create your user account

Configure your Ubuntu distro for the proxy¶

  1. ~/.bashrc gets run everytime you log in to a terminal, so we need to add the following lines to the end of it:

    export http_proxy=http://proxy-chain.intel.com:911
export https_proxy=http://proxy-chain.intel.com:912

  2. APT is the package management software for Ubuntu. We need to create /etc/apt/apt.conf and add the following line to get it to work:

    Acquire::http::Proxy "http://proxy-chain.intel.com:911";

  3. SSH needs to know to use the proxy for github access, so we need to add the following lines to ~/.ssh/config:

    host github.com
port 22
user git
ProxyCommand nc -x proxy-us.intel.com:1080 %h %p

  4. I needed to update permissions on the .ssh/config file:

    chmod 600 ~/.ssh/config
chown $USER ~/.ssh/config

Set up your ssh key¶

  1. From the terminal run the following command, and use the default prompts (having a passphrase is recommended):

    ssh-keygen -t rsa -b 4096 -C "your_email@example.com"

    Output will reflect your environment but be similar to:

    Enter a file in which to save the key (/c/Users/you/.ssh/id_rsa):[Press enter]
Enter passphrase (empty for no passphrase): [Type a passphrase]
Enter same passphrase again: [Type passphrase again]

  2. Now you’ll need to copy the public key to github. Instructions for that are here: https://help.github.com/articles/adding-a-new-ssh-key-to-your-github-account/

Install dependencies¶

  1. We need to get APT up to date before we start adding software packages:

    sudo apt update

  2. Some packages that might be useful for documentation:

    sudo apt install doxygen graphviz default-jre

  3. Python3 should be installed, so now we just need to install pip:

    sudo apt -y install python3-pip

  4. Create a file called requirements.txt and add the following:

    breathe==4.9.1
sphinx==1.7.5
docutils==0.14
sphinx_rtd_theme
sphinxcontrib-plantuml

  5. Run the following command from the terminal to install the above Python modules for Sphinx development:

    pip3 install -r requirements.txt

Some Rules¶

  1. If you want to use a Windows GUI editor (Sublime, for example), then files must be stored on the Windows partition. Files located on your Windows filesystem can be found from the Linux partition here:

    /mnt/c/rest/of/path

    A downside to this is that paths become quite long. You can workaround this by creating a symlink in your Ubuntu home directory:

    ln -s /mnt/c/path/to/your/git_repos ~/work_dir

    And it can be used like this:

    cd work_dir

    Note

    This is important because you must never edit files in the Linux partition from Windows: https://blogs.msdn.microsoft.com/commandline/2016/11/17/do-not-change-linux-files-using-windows-apps-and-tools/
